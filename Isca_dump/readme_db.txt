================================================================================
ИНСТРУКЦИЯ ПО ВОССТАНОВЛЕНИЮ БАЗЫ ДАННЫХ "Isca"
================================================================================

Данный дамп содержит полную структуру и данные базы геоданных для 
гидрологического моделирования водосбора р. Иска (ВКР, ТюмГУ, 2026).

Формат дампа: Plain (SQL)
СУБД: PostgreSQL 16+
Расширение: PostGIS 3.2+ (включая postgis_raster)

================================================================================
ТРЕБОВАНИЯ
================================================================================

Перед восстановлением убедитесь, что у вас установлено:

1. PostgreSQL версии 16 или выше
2. Расширение PostGIS версии 3.2 или выше (ВАЖНО: для PostGIS 3.0+ требуется 
   установка двух расширений: postgis И postgis_raster)
3. pgAdmin 4 (или любое другое средство администрирования PostgreSQL)

================================================================================
ПОШАГОВАЯ ИНСТРУКЦИЯ (через pgAdmin)
================================================================================

ШАГ 1. Создайте пустую базу данных
-----------------------------------
1. Откройте pgAdmin и подключитесь к вашему серверу PostgreSQL.
2. Кликните правой кнопкой мыши на "Databases" и выберите "Create" → "Database...".
3. В поле "Database" введите имя: Isca
4. В поле "Owner" выберите пользователя (обычно postgres).
5. В поле "Encoding" выберите "UTF8".
6. Перейдите на вкладку "Definition":
   - В поле "Template" выберите "template0" (важно для чистой кодировки).
7. Нажмите "Save".

ШАГ 2. Установите расширения PostGIS
------------------------------------
ВАЖНО! Расширения PostGIS должны быть установлены ДО восстановления дампа.
Для версии PostGIS 3.0+ это два разных расширения: postgis и postgis_raster.

ЧЕРЕЗ ГРАФИЧЕСКИЙ ИНТЕРФЕЙС pgAdmin:
------------------------------------
1. В дереве объектов pgAdmin найдите вашу новую базу "Isca".
2. Откройте её, найдите пункт "Extensions".
3. Кликните правой кнопкой мыши на "Extensions" и выберите "Create" → "Extension...".
4. В поле "Name" введите или выберите: postgis
5. В поле "Schema" выберите "public".
6. Нажмите "Save".
7. Повторите шаги 3-6 для расширения: postgis_raster

ЧЕРЕЗ QUERY TOOL (более быстрый способ):
----------------------------------------
1. В pgAdmin откройте Query Tool для базы "Isca" (иконка "Query Tool" в верхней панели).
2. Выполните следующие команды:
   CREATE EXTENSION IF NOT EXISTS postgis;
   CREATE EXTENSION IF NOT EXISTS postgis_raster;
3. Нажмите "Execute" (или F5).

ПРОВЕРКА:
---------
Выполните запрос в Query Tool:
SELECT extname, extversion FROM pg_extension WHERE extname LIKE 'postgis%';

Ожидаемый результат:
postgis        | 3.2.x
postgis_raster | 3.2.x

ПРИМЕЧАНИЕ: Если возникает ошибка "extension 'postgis_raster' is not available", 
значит в вашей системе не установлен пакет postgis-raster. 
Установите его:
- Windows: Используйте Stack Builder при установке PostgreSQL.
- Linux: sudo apt install postgis postgresql-16-postgis-3
- Mac: brew install postgis

ШАГ 3. Восстановите дамп
-------------------------
1. В дереве объектов pgAdmin найдите базу "Isca".
2. Кликните правой кнопкой мыши на "Isca" и выберите "Restore...".
3. В открывшемся окне перейдите на вкладку "General":
   - В поле "Format" выберите "Plain" (это соответствует формату SQL-дампа).
   - В поле "Filename" нажмите на папку и укажите путь к файлу дампа (isca_dump.sql).
4. Перейдите на вкладку "Options":
   - Убедитесь, что галочка "Clean before restore" НЕ установлена 
     (это предотвращает случайное удаление данных).
   - Галочка "Single transaction" может быть установлена (рекомендуется).
5. Нажмите кнопку "Restore".

ШАГ 4. Проверьте успешность восстановления
------------------------------------------
1. После завершения процесса в окне журнала должно отобразиться сообщение 
   "Restore completed successfully".
2. В дереве объектов pgAdmin разверните базу "Isca" → "Schemas" → "public" → "Tables".
3. Убедитесь, что таблица "Landscape" присутствует и содержит данные.
4. Выполните проверочный запрос в Query Tool:
   SELECT COUNT(*) FROM "Landscape";
   Ожидаемый результат: 1176 (количество стокоформирующих комплексов).

ШАГ 5. Проверьте наличие пространственных функций
-------------------------------------------------
Выполните запрос в Query Tool:
SELECT ST_Area(geom) FROM "Landscape" LIMIT 1;

Если запрос возвращает значение площади (число), значит PostGIS работает корректно.

================================================================================
ВОЗМОЖНЫЕ ОШИБКИ И ИХ РЕШЕНИЕ
================================================================================

ОШИБКА 1: "ERROR: extension 'postgis' is not available"
------------------------------------------------------
РЕШЕНИЕ: Установите расширение PostGIS в вашу систему:
- Windows: Используйте Stack Builder при установке PostgreSQL.
- Linux: sudo apt install postgis postgresql-16-postgis-3
- Mac: brew install postgis

ОШИБКА 2: "ERROR: extension 'postgis_raster' is not available"
--------------------------------------------------------------
РЕШЕНИЕ: Это означает, что в вашей системе PostGIS установлен, но без поддержки 
растров. Установите postgis-raster дополнительно:
- Windows: В Stack Builder выберите "PostGIS Raster" компонент.
- Linux: sudo apt install postgresql-16-postgis-3-raster
- Mac: brew install postgis (должен установить всё вместе)

ОШИБКА 3: "ERROR: permission denied for schema public"
------------------------------------------------------
РЕШЕНИЕ: Убедитесь, что вы используете суперпользователя (postgres) 
или имеете права на создание объектов в схеме public.

ОШИБКА 4: "ERROR: relation 'Landscape' already exists"
------------------------------------------------------
РЕШЕНИЕ: Вы уже восстанавливали дамп ранее. Удалите базу "Isca" и создайте 
заново (ШАГ 1) либо выполните восстановление с установленной галочкой 
"Clean before restore" (будьте осторожны!).

ОШИБКА 5: "invalid byte sequence for encoding UTF8"
---------------------------------------------------
РЕШЕНИЕ: Убедитесь, что база данных создана с кодировкой UTF8 
(ШАГ 1, пункт 5). Пересоздайте базу с шаблоном "template0".

ОШИБКА 6: "ERROR: function st_area(geometry) does not exist"
-------------------------------------------------------------
РЕШЕНИЕ: Расширение postgis не было установлено или установлено некорректно. 
Проверьте установленные расширения (ШАГ 2).

================================================================================
АЛЬТЕРНАТИВНЫЙ СПОСОБ (через командную строку)
================================================================================

Для опытных пользователей — восстановление через psql:

1. Откройте командную строку (терминал).
2. Сначала создайте базу и установите расширения:
   psql -U postgres -c "CREATE DATABASE Isca WITH ENCODING='UTF8' TEMPLATE=template0;"
   psql -U postgres -d Isca -c "CREATE EXTENSION postgis;"
   psql -U postgres -d Isca -c "CREATE EXTENSION postgis_raster;"
3. Восстановите дамп:
   psql -U postgres -d Isca -f путь_к_папке/isca_dump.sql

Пример для Windows:
psql -U postgres -d Isca -f C:\Projects\Isca\Database_Dump\isca_dump.sql

================================================================================
ПОДКЛЮЧЕНИЕ К QGIS
================================================================================

После успешного восстановления базу можно подключить к QGIS:

1. Откройте QGIS.
2. В панели Browser найдите "PostgreSQL" и кликните правой кнопкой мыши.
3. Выберите "New Connection...".
4. Заполните поля:
   - Name: Isca (или любое удобное имя)
   - Host: localhost
   - Port: 5432
   - Database: Isca
   - User: postgres
   - Password: [ваш пароль]
5. Нажмите "Test Connection" для проверки.
6. Нажмите "OK".

Теперь слой "Landscape" доступен для работы в QGIS.

================================================================================
ВАЖНОЕ ПРИМЕЧАНИЕ О ВЕРСИЯХ POSTGIS
================================================================================

ВНИМАНИЕ! В PostGIS 3.0 и выше поддержка растров была вынесена в отдельное 
расширение postgis_raster. Если вы используете:
- PostGIS 2.x: расширение postgis включает всё, включая растры.
- PostGIS 3.x+: нужно устанавливать оба расширения: postgis и postgis_raster.

Для проверки версии выполните:
SELECT postgis_version();

Если у вас PostGIS 2.x и вы не можете установить postgis_raster, 
обновитесь до PostGIS 3.x (рекомендуется).
